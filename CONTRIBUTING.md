# Contributing to SecureScan Framework

Thank you for your interest in contributing to the SecureScan Framework! We welcome contributions from the community to help make this project better.

## How to Contribute

### 1. Report Bugs
If you find a bug, please create a new issue describing:
- Steps to reproduce the bug
- Expected behavior
- Actual behavior
- Screenshots (if applicable)

### 2. Suggest Features
We love new ideas! Verification of feature requests happens in GitHub Issues. Please clearly describe the problem you are trying to solve and your proposed solution.

### 3. Submit Pull Requests
1. Fork the repository.
2. Create a new branch (`git checkout -b feature/amazing-feature`).
3. Make your changes.
4. Run tests (`make test`) to ensure everything is working.
5. Commit your changes (`git commit -m 'feat: Add amazing feature'`).
6. Push to the branch (`git push origin feature/amazing-feature`).
7. Open a Pull Request.

## Development Setup

Please refer to the [README.md](README.md) for detailed development setup instructions.

## Testing
We expect all new features to include tests.
- **Backend**: Pytest
- **Frontend**: Jest
- **CLI**: Jest

Run all tests with:
```bash
make test
```

## Code of Conduct
Please be respectful and considerate of others. We aim to foster an inclusive and welcoming community.
